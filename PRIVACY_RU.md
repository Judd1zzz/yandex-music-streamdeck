# Политика конфиденциальности

**Интеграция с «Яндекс Музыка»** (плагин для Stream Deck / StreamDock)

Последнее обновление: 07.07.2026

## Как плагин обрабатывает ваши данные

Плагин **не** собирает, не хранит и не передаёт никаких персональных данных, аналитических данных или телеметрических данных. В нём отсутствуют трекеры и сторонние аналитические сервисы.

## Что остаётся на вашем компьютере

- Настройки плагина (стили кнопок, режим управления, настройки загрузки, если они доступны) хранятся локально в хост-приложении Stream Deck / StreamDock.
- Опциональный токен OAuth Yandex Music (используемый только для экспериментального облачного режима и запросов метаданных треков в сборках, где эти функции присутствуют) хранится локально в глобальных настройках хост-приложения и никогда не отправляется никуда, кроме конечных точек API Yandex Music.

## Сетевые подключения, которые устанавливает плагин

- `api.music.yandex.net` — запросы к API «Яндекс Музыка», отправляемые от вашего имени (метаданные треков; загрузки в сборках, включающих функцию загрузки).
- `avatars.yandex.net` — обложки альбомов для функции «Rich Presence» в Discord.
- `127.0.0.1` (localhost) — связь с настольным приложением «Яндекс Музыка» через протокол Chrome DevTools и с хост-приложением Stream Deck. Данные никогда не покидают ваш компьютер.
- Discord Rich Presence использует локальное IPC-соединение с запущенным клиентом Discord. Данные никогда не покидают ваш компьютер.
- API GitHub Releases (`api.github.com`) — **только в сборках, распространяемых через GitHub/StreamDock**, для проверки наличия обновлений плагина. Сборка из Elgato Marketplace не содержит средства автоматического обновления; обновления предоставляются самим Marketplace.

## Ваша учётная запись

Вход в систему происходит исключительно в официальном приложении «Яндекс Музыка». Плагин никогда не видит и не запрашивает ваш пароль.
